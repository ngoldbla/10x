- Build trains (PR #10 rule): tvOS `cc*10+0`, iOS `cc*10+1`, mac `cc*10+2`. Watch app version rides the iOS archive in lockstep (`MARKETING_VERSION` pinned to app's).
- `CouchScale.chrome`: tvOS 1.0, iOS 0.55, **macOS 0.70**, **watchOS 0.42** (first-guess values per PRDs; tune on screenshot review).
- Never-misfire covenant on every new input grammar (keyboard is trivially safe; right-stick needs 0.75 magnitude + return-to-rest via `FlickClassifier`; Crown commit is always explicit).
- tvOS and iOS behavior must remain byte-identical when the new platform code is inert (no controller connected, non-mac, non-watch builds).
- The Siri Remote's microGamepad belongs to RemoteKit; PadKit filters to `extendedGamepad` only. The two readers must never both claim a device.
- Each phase updates `nine/DEVIATIONS.md` (and `nine/COUCHKIT-ASKS.md` where CouchKit changes) for its own decisions.
- Signing: implement `project.yml`/entitlements/Fastfile config per PRDs, but **do not attempt portal/match minting** from the worktree; record required mint steps in the PR body (PRD-3 ¬ß3 sequencing: portal ‚Üí mint ‚Üí CI).

## Shared contracts (pinned so phases compose)

### TutorialGrammar (built in Phase 4, reused by Phases 5 & 6-adjacent copy)

`nine/Sources/App/TutorialGrammar.swift`, no `#if` (pure data):

```swift
/// Per-platform gesture nouns/verbs for the five tutorial beats
/// (goal ‚Üí place ‚Üí pencil ‚Üí highlight ‚Üí difficulty).
struct TutorialGrammar {
    let placeVerb: String        // "flick", "type", "flick the right stick"
    let pencilVerb: String
    let highlightVerb: String
    let advanceHint: String
    static let remote: TutorialGrammar   // tvOS Siri Remote (existing copy)
    static let touch: TutorialGrammar    // iOS (existing copy)
    static let keyboard: TutorialGrammar // PRD-4
    static let pad: TutorialGrammar      // PRD-5
}
```

`TutorialView` consumes a `grammar` parameter instead of hard-coded copy; existing iOS copy becomes `.touch` verbatim (no copy regressions).

### Afterglow pattern factory (built in Phase 5, iPhone behavior unchanged)

`AfterglowHaptics.swift` splits into:

```swift
enum AfterglowScore {           // #if canImport(CoreHaptics); pure builders
    static func solvePattern() throws -> CHHapticPattern   // exact 9-tick crescendo + 2.40s thump
    static func placementTick() throws -> CHHapticPattern  // whisper transient
    static func errorKnock() throws -> CHHapticPattern     // soft double-knock
    static func boxDetent() throws -> CHHapticPattern      // one detent tick
}
```

iPhone `AfterglowHaptics` keeps its class name/API (`playSolveScore()`, `stop()`) and plays `AfterglowScore.solvePattern()` through `CHHapticEngine`; PadKit plays the same patterns through `GCDeviceHaptics`-vended engines. Timing constants must not drift (verification: solve crescendo identical on iPhone before/after refactor).

### Gate-widening ownership

- Phase 4 (Mac) widens: CouchKit sources for macOS; `GameCenter.swift` ‚Üí `os(iOS) || os(macOS)`; `HistorySheet.swift` + `TutorialView.swift` ‚Üí `+ os(macOS)`; builds `TutorialGrammar`; moves `TouchRose.flickDirection` math to a shared helper.
- Phase 5 (Pad) widens: `GameCenter.swift`/`HistorySheet.swift`/`TutorialView.swift` ‚Üí `+ os(tvOS)`; adds PadKit to CouchKit (`#if os(tvOS)`); refactors AfterglowHaptics into the factory.
- Phase 6 (Watch) widens: CouchKit sources for watchOS; `GameCenter.swift` slim watch branch (auth + report only); does **not** widen HistorySheet/Tutorial to watch.
